Genetic Algorithm Project

As part of my course project, I wrote a program that allow users to build an environment with obstacles and defined path, then train the genetic algorithm to control a charged particle around a path. The genetic algorithm can control the charged particle by varying the potential difference across charged plates. In this document I will be highlighting interesting components of the projects and things that I have learnt during the project. If you are only interested in the general overview of what is involved in the project I will recormend only reading the Algorighm and Lessons Learned sections.

Context

The requirement of the project was to write a python program with a graphical user interface that encorporate a physical concept that was tought in the term. Three weeks were given to carry out the project whils other courses that ran in parallel were held as normal. This meant that time constraint was tight for the project and time management was key aspect of the project.

Controlling a charged particle


Electric and magnetic force that is eperienced by a particle can be given by the following equation. ① F⇀=q×(E⇀+V⇀×B⇀) Where

  • F→ is force experienced by the charged particle
  • q is the charge of the particle
  • E→ is the Electric Field at the point where the particle exist
  • V→ is the Velocity of the particle
  • B→ is the Magnetic Field at the point where the particle exist
As can be seen from equation ①, by varying E field at the point of the particle, Force can be applied to the charged particle in the drsired direction with desired magnitude.

Lessons learned

Technical tips

  • Only use tkinter for tiny projects.
    • Biggest advantage of tkinter is that tkinter is preinstalled in most python enviroments.
    • Many libraries such as MayaVi do not have backend support for tkinter. I would recormend Kivy or PyQT for project that require modern libraries since they would have higher chance of being supported by other libraries.
  • Never use matplotlib for real time three dimensional rendering
    • matplotlib is a library that was initially designed for two dimensional static plots
    • Even though animations and three dimentiaonl plots are supported, the library was not initially designed with these features in mind.
    • As a result, animating three dimensional animation can be computationally costly and some features are not well supported.
    • Use MayaVi for this kind of application since MayaVi was desgined with real time three dimensional plotting in mind.
